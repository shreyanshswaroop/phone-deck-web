## PhoneDeck Mac app list sync

Deck Studio can show real installed Mac apps when the Mac helper sends an app
list through the cloud socket.

Deck Studio emits this when it connects:

```ts
socket.emit("mac-apps-request");
```

The Mac helper or cloud server should answer with either `mac-apps-current` or
`mac-apps-update`:

```ts
socket.emit("mac-apps-current", {
  apps: [
    {
      id: "com.apple.Safari",
      name: "Safari",
      command: "open:com.apple.Safari",
      bundleId: "com.apple.Safari",
      path: "/Applications/Safari.app",
      icon: "data:image/png;base64,..."
    }
  ]
});
```

`icon` should be a browser-loadable URL, preferably a PNG data URL generated by
the Mac helper from the `.app` icon. If no real app list has arrived yet, Deck
Studio keeps showing the built-in sample apps.
